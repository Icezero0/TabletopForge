# 2 用户角色

## 2.1 站点用户

系统中的普通注册用户，可以创建或加入房间。对应 `site_role = user`（站点管理员为 `admin`，不参与跑团身份）。

## 2.2 房间管理者

拥有房间治理权限的成员：`room_role = owner` 或 `manager`。

负责：

- 审批加入房间
- 管理成员
- 修改房间信息
- 设置或解除 manager（仅 owner）

## 2.3 游戏身份（`game_role`）

与 `room_role` **独立**，存储在 `room_members.game_role`。枚举：

```text
GM   # 主持
PL   # 玩家
OB   # 旁观者（Observer）
```

**动机**：房主可 `room_role=owner` 且 `game_role=PL`，无需为当玩家另开房间；`manager` 也可仅为治理者而非 GM。

### GM（`game_role = GM`）

- 地图底图、怪物、任意 Token 与角色 / 状态
- 查看怪物精确 HP
- 绘制层可用

### PL（`game_role = PL`）

- 维护主 PC 与自建附加角色；管理自己创建的角色资源
- 移动绑定自己角色的 Token；绘制层可用
- 怪物仅看 **伤害记录**

### OB（`game_role = OB`）

- 以只读为主（视口、公开信息等），**MVP 可简化为仅实现 GM/PL，OB 列入权限矩阵预留**

## 2.4 room_role 与 game_role 示例

| room_role | game_role | 含义 |
|---|---|---|
| owner | GM | 房主兼主持 |
| owner | PL | 房主兼玩家（常见） |
| manager | PL | 管治理、自己玩 |
| member | PL | 普通玩家 |

# 3 核心使用场景

## 3.1 创建跑团房间

用户创建一个房间，设置名称、简介、规则集和可见性。

## 3.2 加入房间

用户通过房间链接或房间列表申请加入。房间管理者审批通过后，用户成为房间成员。

## 3.3 管理角色卡

玩家创建自己的角色，维护属性、技能、HP、资源、法术位、装备与背景设定。DM 可查看和管理房间内角色。

## 3.4 进行地图协作

DM 创建场景、上传地图、配置网格，并布置玩家、NPC、怪物和物件 Token。房间成员可以在权限允许范围内移动 Token。

## 3.5 聊天与 RP

房间成员可以使用普通聊天进行 OOC 交流，也可以使用 RP 消息以角色身份描述行动和台词。

## 3.6 骰子判定

玩家和 DM 可以进行公开骰、暗骰、技能检定、豁免检定等操作。

## 3.7 状态追踪

系统记录 HP、状态、资源、Token 移动、地图切换等关键变化，并形成操作日志。

# 4 功能范围

## 4.1 已实现（Phase 1 地基）

与代码一致，详见 [`10_repository_status.md`](10_repository_status.md) §4.1：

- 用户注册与登录、资料与头像
- 创建和加入房间、成员审批、`site_role` / `room_role`
- 普通聊天、站内通知、用户反馈
- WebSocket 房间连接与消息同步
- 业务 `assets` 仅头像与反馈图

## 4.2 下一版 MVP（跑团桌面）

在 Phase 1 之上交付，**新增** `room_members.game_role`（`GM` | `PL` | `OB`）。跑团权限 **仅看 game_role**；`room_role` 只管治理。

### 4.2.1 游戏主界面

- 布局与工具见 [`03_frontend_design.md`](03_frontend_design.md) §4.2（地图核心、顶栏工具、左上角色列表与悬浮聊天、底栏素材、右侧信息栏）。
- **MVP**：六个（及场上角色列表）悬浮面板 **锚点固定**，仅可向预设方向收起（§5.10.2；**不可** 自由拖动）。跑团桌面 MVP **完成后** 的可配置布局见 §4.3。
- 右侧 **信息查看单槽**（新替旧）；**个人备忘录**仅自己可见、按房间保存。
- Table 支持 **整体缩放** 与 **小手平移** 视口；Pointer 同步光标与激光指向。
- 多人在线 **实时同步**（权威状态在后端）。

### 4.2.2 地图与网格

- 单房间 **单活跃地图桌面**（多场景切换后续）。
- 图层：地图底图基准 **z=0**，Token 基准 **z=100**，绘制基准 **z=200**；同类内可排序（见 `tabletop_scene.md` §3.2）；**网格线始终显示**。
- GM 上传地图素材，**缩放适配网格**；可 **锁定** 底图（锁定后不可再移动底图）。
- 底图删除：选中 → 右键菜单删除。
- **测距**：**MVP 不做**（后续：Token 轨迹测距、困难地形分段等，见 `tabletop_scene.md` §3.5）。
- **自定义绘制**：笔刷、线、方、圆、**文本框**（字号、颜色）；线宽与颜色；橡皮擦与框选删除；**GM 与 PL** 可绘制。
- 细节见 [`09_module_design/tabletop_scene.md`](09_module_design/tabletop_scene.md)。

### 4.2.3 Token

- 角色 Token、怪物 Token（类型对应图片 + 信息）。
- 添加、编辑、拖拽、**Token 类内** z-index、缩放。
- 删除：选中 → 右键菜单删除。
- 角色 Token：地图 HP 预览，点击查看角色详情。
- 怪物：GM 看全量 HP 与详情；PL 仅看 **累计受伤害 / 伤害记录**。
- 细节见 [`09_module_design/token_system.md`](09_module_design/token_system.md)。

### 4.2.4 角色卡与实时状态

**两层模型**（见 `character_card.md`）：

| 层 | 内容 |
|---|---|
| **Character（稳定）** | 名称、职业等级、六维、技能（手填）、豁免熟练摘要、自定义 KV；RP **第一页 / 第二页** 大文本；**不做** 属性/技能自动计算 |
| **CharacterState（实时）** | 当前/最大 HP、实时 AC、临时 Buff 等 |

- **主 PC / 附加角色 / 怪物**：预览用结构化字段 + 大文本；附加角色与怪物可更简。
- **自定义 key/value**：用户自增字段。
- PL 管理 **自己创建** 的角色资源；GM 管理全部。
- **Token 图**可选；无图用 **名称首字**。

### 4.2.5 怪物与复用

- GM 创建怪物：稳定层 + 状态层；正文以大文本为主。
- 房间内 **复用** 同一 `character_id` 多 Token。
- PL 不可改 GM 怪物正文；HP 见 Token 可见性规则。

### 4.2.6 MVP 依赖与横切

- `game_role` 字段与迁移；入房默认 `game_role` 策略（建议默认 `PL`，房主可选 `GM`）。
- 业务 assets（地图、Token 图）；操作日志。
- 权限：`08_permission_design.md` §6.4。

## 4.3 后续增强功能

- **跑团主界面悬浮面板布局**（跑团桌面 MVP **完成后**）：
  - MVP：六个（及场上角色列表）悬浮区 **位置固定**，仅向预设方向收起（见 `03_frontend_design.md` §4.2.8）。
  - 后续：从 **统一入口** 勾选要显示的面板；已显示面板可在视口上 **自由拖动**；布局按用户（+ 可选按房间）持久化，默认各用户互不同步。
- **Campaign / Session / Scene**：团务时间线 — Campaign 为长线故事，Session 为单次场次，Scene 与地图对应；用于历史与多场景（**非 MVP**）
- Token **移动测距**、困难地形、可达判定
- 属性 / 技能 **自动计算** 与规则效果
- 多场景切换
- 骰子、RP 结构化消息、DND5E 自动计算
- 战斗先攻、回合管理
- 角色卡导入导出、怪物模板
- 房间级资源库分类

## 4.4 暂不实现功能

- 动态光照、战争迷雾
- 完整法术数据库、职业升级自动化、自动战斗结算
- 宏系统、多规则集完整实现
- `game_role=OB` 完整体验（可先预留枚举）
- Campaign / Session / Scene 归档
- 测距与 Token 移动规则引擎

### 5.10.4 工具模式与行为

- **模式互斥**：手型 / 绘制 / Pointer（激光按下时）不触发选中框；**测距不在 MVP**。
- **选择模式**（默认或独立「箭头」工具）：单击可选中；**新选中替换旧选中**。
- **选中态**：外部边框 + 缩放柄 + Context Menu。
- **InfoPanel**：仅 **一条** 查看信息；新打开 **替换** 旧内容；关闭后可为空。
- **PersonalMemo**：按房间保存、仅自己可见；收起状态仅本地或轻量同步 UI 态。

### 5.10.5 交互流

**进入房间**

1. 路由 `/rooms/:id` → 鉴权 → 加载房间与成员身份。
2. 建立 WebSocket → 同步消息、成员、后续地图 / Token / 绘制事件（◎）。
3. 默认 **选择模式**；视口与备忘录恢复上次（◎ 可选）。

**添加地图（GM）**

1. 底栏「添加地图」→ 选图上传 → 底图出现在地图 band（基准 z=0）。
2. GM 缩放 / 拖拽对齐网格 → 可选「锁定」。

**添加角色**

1. 底栏「添加角色」→ 类型：主 PC / 附加角色 / 怪物 → 稳定层 + 状态层表单（六维技能手填；RP 第 1/2 页大文本；可自定义 KV）。
2. 写入房间角色库 → 底栏列表出现条目。

**角色上场**

1. 左上列表点「上场」或底栏拖拽 → 在地图创建 Token（绑定 `character_id`）。
2. 无图则 **名称首字** 显示。

**查看 Token 信息**

1. 选择模式 → 单击 Token → 菜单「查看信息」。
2. 右侧 InfoPanel 展示（怪物：实例名、已受伤害、详情；PL：按权限字段）。
3. 再对另一 Token 查看信息 → **替换** InfoPanel 内容。

**绘制标注**

1. 切换绘制工具 → 选子工具（线 / 方 / 圆 / 文本框）与颜色线宽字号。
2. 在地图上绘制 → 持久化并广播。
3. 橡皮擦或框选 → 删除绘制对象。

**Pointer**

1. 常驻同步他用户光标与标签。
2. 按住指针 → 激光指示 → 松手消失。

**聊天**

1. 左上悬浮面板输入 → 发送 → 列表更新 + WS 广播（✓）。

**离开 / 断线**

1. 离开路由关闭 WS；重连后重新拉取房间状态与视口（◎）。

# 6 成功标准

### Phase 1（已实现）

1. 用户可以注册、登录、创建 / 加入房间并完成成员审批。
2. 房间内可以普通聊天，关键事件经 WebSocket 同步。
3. `site_role` 与 `room_role` 权限在后端可校验。

### 跑团桌面 MVP（目标）

1. `game_role` 与 `room_role` 分离；房主可作为 PL 跑团。
2. GM 可管理地图与怪物；PL 可管理自建角色；角色稳定层与状态层分设。
3. 角色无 Token 图时显示名称首字；角色定义可在房间内复用。
4. 绘制、地图锁定、右键删除等符合 `08` §6.4；**不测距**。
5. Table 操作经 HTTP + WebSocket 同步；Campaign/Session/Scene 留后续。
