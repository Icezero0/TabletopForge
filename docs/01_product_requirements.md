# TabletopForge 产品需求文档

版本：v0.2  
状态：Draft

---

# 1 文档定位

本文档描述 TabletopForge 的产品目标、用户角色、核心场景、功能范围与阶段规划。

本文档关注“要做什么”和“为什么做”，不展开具体技术实现。

---

# 2 用户角色

## 2.1 站点用户

系统中的普通注册用户，可以创建或加入房间。

## 2.2 房间管理者

拥有房间治理权限的成员，包括房主和房间管理员。

负责：

- 审批加入房间
- 管理成员
- 处理房间身份变更
- 修改房间设置

## 2.3 DM

游戏主持者，负责游戏内容管理。

负责：

- 创建和切换场景
- 上传地图
- 创建 NPC、怪物和隐藏对象
- 管理战斗
- 修改游戏内状态
- 发起暗骰

## 2.4 玩家

参与游戏的成员，通常拥有自己的角色。

负责：

- 维护自己的角色卡
- 控制自己的角色 Token
- 发送普通聊天与 RP 消息
- 进行骰子检定

## 2.5 旁观者

仅观看房间内容或有限参与交流的成员。

旁观者默认不具备游戏内编辑权限。

# 3 核心使用场景

## 3.1 创建跑团房间

用户创建一个房间，设置名称、简介、规则集和可见性。

## 3.2 加入房间

用户通过房间链接或房间列表申请加入。房间管理者审批通过后，用户成为房间成员。

## 3.3 管理角色卡

玩家创建自己的角色，维护属性、技能、HP、资源、法术位、装备与背景设定。DM 可查看和管理房间内角色。

## 3.4 进行地图协作

DM 创建场景、上传地图、配置网格，并布置玩家、NPC、怪物和物件 Token。房间成员可以在权限允许范围内移动 Token。

## 3.5 聊天与 RP

房间成员可以使用普通聊天进行 OOC 交流，也可以使用 RP 消息以角色身份描述行动和台词。

## 3.6 骰子判定

玩家和 DM 可以进行公开骰、暗骰、技能检定、豁免检定等操作。

## 3.7 状态追踪

系统记录 HP、状态、资源、Token 移动、地图切换等关键变化，并形成操作日志。

# 4 功能范围

## 4.1 MVP 功能

MVP 应优先覆盖以下功能：

- 用户注册与登录
- 创建和加入房间
- 成员审批
- 三层身份模型
- 普通聊天
- 角色卡基础信息
- 角色状态管理
- DND5E 基础计算
- 骰子表达式
- 地图场景基础能力
- Token 创建和移动
- 资源上传
- 操作日志
- WebSocket 实时同步

## 4.2 后续增强功能

- 战斗先攻列表
- 回合与轮数管理
- 状态持续时间
- 暗骰可见性控制
- RP 记录导出
- 房间级资源分类
- 角色卡导入导出
- 简单怪物模板
- 自定义规则集字段

## 4.3 暂不实现功能

- 动态光照
- 战争迷雾
- 完整法术数据库
- 完整职业升级自动化
- 自动战斗结算
- 宏系统
- 复杂权限继承
- 多规则集完整实现

# 5 页面需求

## 5.1 登录 / 注册页面

提供基本认证入口。

## 5.2 房间列表页面

展示用户创建或加入的房间。

## 5.3 房间详情 / 主工作台页面

核心业务页面，包括：

- Table 区域
- 工作区 tabs
- 成员状态
- 当前场景
- 骰子入口
- 房间设置入口

## 5.4 角色详情页面

展示和编辑单个角色的角色卡、状态、资源、装备和背景。

## 5.5 资源库页面或面板

管理用户资源和房间资源。

## 5.6 管理页面

用于房间成员、审批、身份、反馈等管理功能。

---

# 6 成功标准

初始版本完成后，应满足以下标准：

1. 一个跑团房间可以完成成员加入、角色创建、聊天、RP、骰子和基础地图操作。
2. 角色 HP、状态、资源等变化可以被实时同步并记录日志。
3. DM 可以布置地图和 Token，玩家可以控制自己的角色 Token。
4. 权限系统能够区分站点、房间和游戏三个层级。
5. 文档、接口和数据模型能够支持后续扩展。
