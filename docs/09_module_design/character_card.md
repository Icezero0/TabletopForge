# 角色卡模块设计

版本：v0.3  
状态：Draft

---

# 1 模块目标

角色卡模块用于维护房间内玩家角色、NPC 与怪物的基础信息、规则数据、当前状态与背景设定。

第一阶段优先支持 DND5E 角色卡的基础字段和状态追踪。

---

# 2 核心概念

## 2.1 Character

房间内可复用的**角色定义**（角色资源），用于绑定 Token、编辑信息与多次上场。类型包括：

- 玩家主 PC
- 玩家附加角色（如德鲁伊变身、召唤物等）
- 怪物 / NPC（多由 DM 创建）

## 2.2 CharacterLibraryEntry（房间角色库）

同一房间内，角色定义进入**房间角色库**，供复用：

- **DM 创建的怪物**等资源：房间级共享，可多次生成 Token 上场。
- **玩家创建的角色**：创建者拥有管理与查看权；是否允许他人复用放置 Token 见 §6（MVP 默认：玩家附加角色仅创建者放置 Token；DM 怪物全员可见、由 DM 放置 Token）。

## 2.3 CharacterState

角色当前状态，例如 HP、临时 HP、AC、状态效果等。

## 2.4 CharacterResource

角色资源，例如法术位、职业资源、短休资源、长休资源等（长期扩展）。

每条角色定义记录 **`created_by_user_id`**（创建者），用于权限与「自己上传的角色资源」范围判定。

# 3 数据范围

## 3.1 玩家主 PC（member）

**核心字段（MVP 结构化）**：

- 名称
- 职业与等级
- HP（当前 / 最大等）
- AC
- 被动察觉（PP）
- 豁免
- 熟练项

**其他区块（MVP）**：BUFF、职业能力、法术、装备、物品等 → **长文本 / 备注块**。

## 3.2 玩家附加角色（member，如变身、召唤等）

除主 PC 外，玩家**可自行添加**更多角色定义，用于短期形态或分身：

**核心字段（MVP 结构化）**：名称、HP、AC。

**其余信息**：**单一大文本框**（与怪物 MVP 形态类似，不要求职业等级等完整卡）。

创建者对该定义拥有查看、编辑、删除及从库中**复用生成 Token** 的权限（见 §6）。

## 3.3 怪物 / DM 角色（owner / manager）

- 信息 MVP 以 **单一大文本框** 为主；可同时维护名称、HP、AC 等核心字段。
- 存入房间角色库，**可在房间内反复复用**（多次创建 Token 绑定同一 `character_id`）。
- 精确 HP 仅 DM 可见；member 见 `token_system.md` §5。

## 3.4 Token 图片与缺省头像

- 角色定义可关联可选 **Token 图片**（业务 asset，由创建者上传）。
- **未上传 Token 图** 时：地图与列表上使用 **名称首字** 作为缺省头像（字形 / 圆形底字）。

## 3.5 长期完整角色卡

后续可扩展：属性、技能、资源、装备、法术位、背景、公开 / 私密分区等（见原 v0.2 范围）。

---

# 4 前端交互

- **房间角色库**列表：展示可复用角色定义（区分创建者、类型）。
- **从库复用**：选择已有定义 → 在地图上生成新 Token（绑定同一 `character_id`）。
- 角色详情 / 编辑（按权限）
- HP 快速修改
- 上传或更换 Token 图；无图时预览首字缺省头像

---

# 5 后端服务

后端服务负责：

- 创建角色
- 更新角色基础信息
- 更新角色状态
- 更新资源
- 写入操作日志
- 发布实时事件

---

# 6 权限规则

MVP 通过 `room_role` 判定（不新增 `game_role`）：

- `owner` / `manager`：创建 / 编辑 / 删除房间内任意角色定义；怪物等资源房间级复用；为任意定义放置 Token。
- `member`：
  - 创建 / 编辑 / 删除 **自己创建** 的角色定义（含主 PC 与附加角色）。
  - **查看与控制**自己上传（创建）的全部角色资源（含附加角色与对应 Token）。
  - 不可编辑 DM 创建的怪物正文；怪物 HP 见 `token_system.md`。
  - 从库复用：仅对自己创建的定义生成 Token；不可新建绑定 DM 怪物库的 Token。
- 旁观者（VIEWER）不在 MVP 范围。
