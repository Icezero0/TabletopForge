# 角色卡模块设计

版本：v0.4  
状态：Draft

---

# 1 模块目标

维护房间内 **角色定义**（较长时间不变）与 **角色实时状态**（对局内频繁变化），支持玩家主 PC、附加角色（变身等）、怪物 / NPC，并与 Token、房间角色库复用联动。

---

# 2 核心概念

## 2.1 Character（角色卡 / 稳定层）

房间内可复用的角色定义，用于预览、编辑与绑定 Token。内容 **不** 承载每轮变动的 HP/AC（见 CharacterState）。

## 2.2 CharacterState（实时状态层）

与 Character 1:1（或按对局会话 1:1，实现时定），存储：

- 当前 HP / 最大 HP、实时 AC
- 临时 BUFF / 状态效果
- 其它对局内频繁变更字段

角色卡改背景、六维；状态改受伤、加 Buff — **分表 / 分 API**。

## 2.3 CharacterLibraryEntry（房间角色库）

同一房间内角色定义可复用：GM 怪物、PL/DM 添加的定义均可多次生成 Token（`character_id` 相同）。

## 2.4 自定义字段

提供 **用户自定义 key/value**（JSON 或 EAV），弥补固定字段不足。

每条角色记录 **`created_by_user_id`**，用于「自己上传的角色资源」权限。

# 3 Character 数据范围（MVP）

## 3.1 结构化预览字段（方便列表 / Inspector 摘要）

主 PC 等需要 **重点展示** 的抽取字段（示例，实现可微调）：

- 名称、职业与等级
- 六维属性（STR/DEX/CON/INT/WIS/CHA）— **手填，MVP 不做自动计算**
- 技能列表 — **手填，MVP 不做技能加值 / 规则自动结算**（含复杂技能效果后续迭代）
- 被动察觉（PP）、豁免、熟练项 — 可手填摘要

**自动计算**（属性调整值、技能加值、法术 DC 等）：**不在 MVP**，避免规则引擎范围膨胀。

## 3.2 大文本区块（参考 TXT 卡第 1 / 2 页）

| 区块 | 内容示例 |
|---|---|
| **第一页（RP 设定）** | 种族、阵营、年龄、性别、背景、外貌等扮演向设定 |
| **第二页（规则长文）** | 职业能力、法术、装备、物品、BUFF 说明等 — 统一 **超大文本框** |

附加角色、怪物：名称 + 少量预览字段 + **单一大文本框** 即可。

## 3.3 自定义 key/value

用户可自行添加字段（如「灵感」「房规备注」），与固定字段并存。

## 3.4 Token 图片

可选 `token_image` asset；无图时 **名称首字** 作缺省头像。

# 4 CharacterState 数据范围（MVP）

- `current_hp`, `max_hp`
- `ac`（实时 AC，含 Buff 后由 GM/PL 手改）
- 临时效果 / Buff 列表 — MVP 可用 **大文本或轻量列表**，不做复杂持续时间引擎
- 怪物 **精确 HP** 仅 GM 可见；PL 见 **伤害记录 / 累计受伤害**（见 `token_system.md`）

Inspector / 地图 HP 预览读 **CharacterState**，不读稳定层里的过时 HP。

---

# 5 前端交互

- 房间角色库：列表预览（名称 + 状态摘要）
- 从库复用 → 新建 Token
- 编辑：稳定层表单 + 状态层分区（或 Tab）
- 上传 Token 图；无图首字预览

---

# 6 权限规则

由 **`game_role`** + `created_by_user_id` 判定（见 `08_permission_design.md` §6.4）：

- **GM**：任意角色稳定层与状态层。
- **PL**：自己 **创建** 的角色；不可改 GM 怪物正文；怪物 HP 可见性受限。
- **OB**：只读（按矩阵）。

`room_role` 仅影响能否审批、踢人等，**不** 替代 GM/PL。

---

# 7 实时事件

- CHARACTER_CREATED / UPDATED / DELETED
- CHARACTER_STATE_CHANGED（HP、AC、Buff 等）
- 与 Token 事件配合
