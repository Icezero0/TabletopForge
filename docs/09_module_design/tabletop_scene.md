# 场景与地图模块设计

版本：v0.3  
状态：Draft

---

# 1 模块目标

场景与地图模块用于维护房间内游戏桌面：背景地图、固定网格、绘制标注、视口与测距。

---

# 2 核心概念

## 2.1 Scene（长期）

一个房间中的游戏场景，例如酒馆、森林、地下城房间。多场景切换为后续能力。

## 2.2 Map

场景中的背景地图对象，含图片资源、位置、缩放、锁定状态。

## 2.3 Grid

**始终显示**的方格网格 UI 叠加层，用于对齐与距离（ft）计算。地图素材导入后通过缩放适配网格，无需单独两点标定比例尺流程。

## 2.4 Drawing

房间成员在绘制层上的标注（笔刷、轴对齐方形、圆形等），单位 ft。

---

# 3 MVP 范围

## 3.1 单活跃桌面

MVP 每个房间 **一个活跃地图桌面**，不实现多场景列表与切换。长期多场景见 §6。

## 3.2 图层与 z-index

固定三层业务 z-index，网格为 UI 叠加（不参与业务层排序）：

| 层 | 默认 z-index | 说明 |
|---|---|---|
| 地图底图 | 0 | 可锁定；锁定后场景坐标中位置不可再移动 |
| Token | 1 | 同层内可调整相对顺序 |
| 自定义绘制 | 2 | 笔刷 / 方形 / 圆形等 |

「图层调整」指 **同层内** 对象叠放顺序，不改变三层相对关系。

## 3.3 地图底图

- DM（`owner` / `manager`）上传地图素材。
- 导入后 **缩放适配网格**。
- 未锁定时可拖动底图对齐网格。
- **锁定**后底图位置不可再移动（视口仍可平移缩放）。
- 删除：选中底图 → **右键 Context Menu → 删除**。

## 3.4 视口

Table 游戏区域支持：

- **缩放**
- **小手平移**（pan）视角

视口变换不改变对象在场景中的权威坐标。

## 3.5 测距工具

- 独立工具模式：激活后 **从一点拖到另一点** 显示直线距离（ft，基于网格尺度）。
- 测距结果 **全员可见**。
- 测距线为临时 UI，切换工具或清除后消失（不写入绘制层持久对象）。

## 3.6 自定义绘制

- 笔刷、方形、圆形（可设半径，单位 ft）。
- 持久化并经 WebSocket 同步。
- **所有房间成员**（`owner` / `manager` / `member`）均可创建、擦除（橡皮擦）与框选删除。
- 删除方式：**橡皮擦**、**框选范围删除**（不用 Context Menu 作为主路径）。

## 3.7 网格

- 方格网格 **始终显示**。
- 每格代表固定 ft（房间或地图级配置，实现时落库）。

# 4 权限规则

权限通过 `room_role` 判定，见 `08_permission_design.md` §6.4：

- `owner` / `manager`：地图上传、锁定、删除底图；绘制与测距。
- `member`：只读地图底图（不可上传 / 删除 / 移动未锁定底图）；**可绘制**、擦除、框选删除；可使用测距工具与视口平移缩放。

---

# 5 实时事件

- MAP_ADDED / MAP_UPDATED / MAP_LOCKED / MAP_DELETED
- DRAWING_CREATED / DRAWING_UPDATED / DRAWING_DELETED（含批量框删）
- MEASURE_OVERLAY（可选，若测距结果需广播；否则仅本地 UI）

长期多场景时保留：SCENE_CREATED、SCENE_CHANGED 等。

# 6 后续扩展

---

# 7 设计原则

1. 地图配置与 Token 数据分离。
2. 网格为对齐与测距基准，底图缩放适配网格。
3. HTTP 持久化 + WebSocket 广播确认后状态。
4. 底图与 Token 删除统一走选中 + 右键菜单；绘制走橡皮擦 / 框选。
