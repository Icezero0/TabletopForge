# 操作日志模块设计

版本：v0.2  
状态：Draft

---

# 1 模块目标

操作日志模块用于记录房间内关键状态变化，使跑团过程可追溯。

---

# 2 记录范围

需要记录：

- HP 修改
- 角色状态变化
- 资源消耗与恢复
- 法术位变化
- 角色卡关键字段修改
- Token 创建、移动、删除
- 场景切换
- 成员身份变化
- 骰子结果
- 战斗状态变化

---

# 3 日志字段

推荐字段：

```text
room_id
operator_user_id
entity_type
entity_id
action_type
before_value
after_value
description
created_at
```

---

# 4 查询能力

支持按以下条件筛选：

- 操作类型
- 操作者
- 角色
- Token
- 时间范围

---

# 5 设计原则

1. 日志应由后端服务层写入。
2. 关键状态变化必须写日志。
3. 日志不应被普通成员删除。
4. 日志用于回溯，不用于替代业务状态表。
