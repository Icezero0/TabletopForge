# Asset 与资源库模块设计

版本：v0.2  
状态：Draft

---

# 1 模块目标

当前后端地基中的 `assets` 模块用于管理与业务无关的基础图片资产，例如用户头像和反馈图片。

用户长期资源库、房间资源库、地图背景和 Token 素材属于后续业务资源库能力，不在当前地基中实现。

---

# 2 当前资产类型

当前实现的 `asset_type`：

- `avatar`：用户头像。
- `feedback_image`：反馈图片。

后续业务资源库可在 `image` 类型下继续细分用途，例如 map、token、handout、portrait、scene 等，但这些不应在基础 assets 阶段提前落表。

---

# 3 资源范围

## 3.1 用户头像

用户上传头像后，用户表保存当前 `avatar_asset_id`。

## 3.2 反馈图片

反馈图片通过 `feedback_id` 关联到反馈记录。

## 3.3 后续业务资源库

长期业务资源库后续再设计：

- 用户长期图片
- 房间图片
- 地图背景
- Token 图片
- handout / portrait / scene 等业务用途

---

# 4 当前功能范围

- 上传头像。
- 创建反馈时上传反馈图片。
- 读取 asset 内容。
- 校验图片类型和大小。

当前暂不实现资源重命名、删除、分类、搜索、房间资源、从资源创建 Token。

---

# 5 权限规则

- avatar：登录用户可读取；当前用户可更新自己的头像。
- feedback_image：反馈创建者可读取；具备查看全部反馈权限的 admin 可读取。
- 业务资源库权限后续随业务资源库一起设计。
