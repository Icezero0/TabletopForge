# Asset 与资源库模块设计

版本：v0.2  
状态：Draft

---

# 1 模块目标

当前后端地基中的 `assets` 模块用于管理底层文件资产，例如用户头像、反馈图片、用户资源库图片和音频。

当前已实现用户级底层资源库 asset；房间资源库、地图背景和 Token 等游戏语义层模型后续再实现。

---

# 2 当前资产类型

当前实现的 `asset_type`：

- `avatar`：用户头像。
- `feedback_image`：反馈图片。
- `image`：用户资源库中的通用图片。
- `audio`：用户资源库中的通用音频。

跑团桌面 MVP 拟增加业务用途，例如：

- `map_background`：地图底图
- `token_image`：角色 / 怪物 Token 图（由创建者上传，挂在角色定义上）

未上传 Token 图时，前端使用角色**名称首字**渲染，不强制落库 asset。

# 3 资源范围

## 3.1 用户头像

用户上传头像后，用户表保存当前 `avatar_asset_id`。

同时，`user_avatar_history` 记录用户上传过的头像 asset。当前头像切换只更新 `users.avatar_asset_id`；旧头像因为仍存在历史记录，不会被释放。后续用户删除历史头像记录时，再释放对应 avatar asset 的一次引用。

## 3.2 反馈图片

反馈图片通过 `feedback_id` 关联到反馈记录。

## 3.3 后续游戏语义资源库

游戏语义层资源库后续再设计：

- 用户长期图片
- 房间图片
- 地图背景
- Token 图片
- handout / portrait / scene 等业务用途

# 4 当前功能范围

- 上传头像。
- 分页读取当前用户的历史头像。
- 创建反馈时上传反馈图片。
- 读取 asset 内容。
- 校验图片类型和大小。
- 上传、列表、读取、删除用户资源库 `image` / `audio`。
- 上传时计算 SHA-256，并按 `asset_type + content_hash + size_bytes + content_type` 复用已有文件。当前 `avatar`、`image`、`audio` 会参与去重，`feedback_image` 因绑定反馈可见性而保持独立。
- 通过 `ref_count` 记录底层文件引用数；删除用户资源库 asset 或历史头像记录时表示释放一次引用，归零才删除文件。

当前暂不实现资源重命名、分类、搜索、房间资源、从资源创建 Token。

---

# 5 权限规则

- avatar：公开可读取；当前用户可更新自己的头像。
- feedback_image：反馈创建者可读取；具备查看全部反馈权限的 admin 可读取。
- image/audio：登录用户可上传和释放引用；内容公开可读取，供共同游戏使用。
- 房间资源和游戏语义资源权限后续随业务模块一起设计。
