# Step 5 Phase 1：Token 基础

**目标**：地图上可出现、可协作操作的 Token；默认 **圆形、直径 = 1 格 = 5 ft**；暂不强制绑定角色。

**依据**：[`05-character-core.md`](./05-character-core.md)、`docs/09_module_design/token_system.md`、`docs/08` §6.4。

**前置**：Step 4 地图/绘制/WS 已落地（`room_tabletop_*`、MapViewport、SelectionOverlay、ContextMenu）。

---

## 数据与 API

### Migration `room_tokens`

MVP 扁平 room 级，**不设** `scene_id`。

```text
id
room_id              FK → rooms.id
asset_id             FK → assets.id, nullable（token_image）
linked_character_id  FK → characters.id, nullable（Phase 1 不校验）
name                 VARCHAR(255)
token_type           VARCHAR — character | monster | object（MVP 默认 character）
x, y                 float，场景坐标
width, height        float，默认 = room grid_cell_ft（1 格直径）
rotation             float，默认 0
z_index              int，Token 类内序号
visible              bool，默认 true
locked               bool，默认 false
owner_user_id        FK → users.id
created_at, updated_at
```

- **z_index 语义**：Token band 类内序号；effective z = **100 + z_index**（见 `tabletop_scene.md` §3.2）。
- 常量 `TOKEN_BAND_BASE = 100` 加入 [`backend/app/modules/rooms/tabletop/constants.py`](../backend/app/modules/rooms/tabletop/constants.py)。

### AssetType

- 新增 `TOKEN_IMAGE`（[`backend/app/modules/assets/constants.py`](../backend/app/modules/assets/constants.py)）。

### HTTP API

扩展 [`backend/app/modules/rooms/tabletop/`](../backend/app/modules/rooms/tabletop/)（复用 `RoomTabletopService` 模式）：

| 方法 | 路径 | 说明 |
|---|---|---|
| GET | `/rooms/{id}/tabletop` | 快照增加 `tokens: []` |
| POST | `/rooms/{id}/tokens` | multipart：可选图 + `name`；默认尺寸 = `grid_cell_ft` |
| PATCH | `/rooms/{id}/tokens/{token_id}` | x/y/width/height/z_index/name/locked 等 |
| DELETE | `/rooms/{id}/tokens/{token_id}` | 删除 Token |

**权限**（消费 [`game_permissions.py`](../backend/app/modules/rooms/game_permissions.py)）：

- GM：`MANAGE_ANY_TOKEN`、`MOVE_ANY_TOKEN` — 任意 Token CRUD/移动。
- PL：`MOVE_OWN_CHARACTER_TOKEN` — Phase 1 按 `owner_user_id === self` 判定（未绑角色时即创建者）。
- OB：只读，PATCH/DELETE → 403。

### WebSocket

扩展 [`backend/app/realtime/publisher.py`](../backend/app/realtime/publisher.py) 与前端 [`useRoomRealtimeSession`](../frontend/src/features/room/composables/useRoomRealtimeSession.ts)：

- `token_created` / `token_updated` / `token_deleted`

### API 测试

`backend/tests/api/test_room_tokens.py`：

- GM 创建 Token（有图 / 无图首字）
- PL 创建自己的 Token；PL 不可改 GM 的 Token
- PATCH 位置与 scale；DELETE
- WS 广播断言

## 本 Phase 做

### 后端

- Alembic migration `room_tokens`
- ORM model、repository、schemas、service、rooms API 路由
- `GET tabletop` 快照含 tokens
- WS 事件与 tabletop store 合并逻辑

### 前端

| 项 | 说明 |
|---|---|
| **Store / API** | 扩展 [`tabletop.store.ts`](../frontend/src/stores/tabletop.store.ts)、[`rooms.api.ts`](../frontend/src/infra/api/rooms.api.ts) |
| **TokenLayer** | 新建 `TokenLayer.vue` + `TokenItem.vue`，插入 [`MapViewport.vue`](../frontend/src/features/table/components/MapViewport.vue)（MapLayer 与 DrawingLayer 之间） |
| **圆形渲染** | `border-radius: 50%` + `overflow: hidden`；无图 → **名称首字** |
| **默认尺寸** | `width = height = settings.grid_cell_ft`（场景坐标，1 格 = 5 ft） |
| **选中交互** | 扩展 [`TabletopSelection`](../frontend/src/features/table/types.ts) → `{ type: 'token', id }` |
| **拖拽/缩放** | `TokenSelectionOverlay.vue`（或泛化 SelectionOverlay）：拖拽 x/y、角点等比缩放 |
| **Context Menu** | 扩展 [`ContextMenu.vue`](../frontend/src/features/table/components/ContextMenu.vue)：**删除**、**图层**（类内 z_index，禁止跨 band） |
| **底栏** | [`BottomAssetBar`](../frontend/src/features/table/components/BottomAssetBar.vue)「添加角色」→ Phase 1 **「放置 Token」**：选图或输入名称（首字）+ 点击地图放置 |

## 验收清单

- [x] `GET /rooms/{id}/tabletop` 返回 `settings + maps + drawings + tokens`。
- [x] GM/PL 可 POST Token（可选图）；无图显示名称首字；默认直径 5 ft（1 格）。
- [x] 选择模式：单击选中 Token → 边框 + 缩放柄；拖拽、角点等比缩放。
- [x] 右键菜单：删除、图层（类内 z_index 上/下/顶/底）；不与地图/绘制 band 混排。
- [x] 变更后 WS 广播 `token_*`；全员渲染一致；刷新从 snapshot 恢复。
- [x] OB 只读渲染，不可 PATCH/DELETE Token。
- [x] 与 Step 4 手型/绘制/Pointer/测距工具互斥无冲突。
