# Step 4：地图与绘制核心

**目标**：中央 MapViewport 具备 **可协作的地图桌面**：底图、网格对齐、视口、锁定、图层、绘制。本步 **不** 做 Token、角色卡、测距。

**依据**：`docs/09_module_design/tabletop_scene.md`、`docs/08` §6.4、`docs/01` §4.2.2。

---

## 数据与 API（概要）

- 表/模型（名称实现时定）：房间 **单活跃** 地图实例（位置、缩放、锁定、asset_id、**地图类内** z-index，band 基准 0）。
- 绘制对象表：笔刷/线/方/圆/文本，绘制 band（基准 **z=200**），归属房间；**绘制类内** z-index。
- 业务 `asset` 类型：`map_background`（扩展 `assets` 模块）。
- HTTP 持久化 + WS 广播（MAP_*、DRAWING_*）。

---

## 本步做

| 功能 | 边界 |
|---|---|
| **导入地图** | GM 从底栏上传；底图在地图 band（基准 z=0）；缩放 **适配网格** |
| **视口** | 手型：平移、缩放整个画布 |
| **地图拖拽/缩放** | 未锁定时可拖、可缩放；**锁定**后位置固定 |
| **地图图层** | **地图类内** z-index（菜单只调同类，见 `tabletop_scene.md` §3.2.3）；Context Menu：删除 / 锁定 / 图层 |
| **网格** | 常显；房间或地图级「每格 ft」配置 |
| **自定义绘制** | GM/PL；笔刷、线、方、圆、文本框（字号/颜色）；线宽/颜色；橡皮擦、框选删 |
| **选中交互** | 选择模式：单击 → 边框 + 缩放柄 + 菜单（地图） |
| **Pointer** | 可选本步或 Step 5：多用户光标 + 按住激光 |

### 权限

- 一律 `game_role`（`08` §6.4），与 `room_role` 无关。

## 验收清单

- [ ] GM 上传地图后全员可见并对齐网格。
- [ ] 锁定后 GM 不可再拖底图；视口仍可 pan/zoom。
- [ ] PL 可绘制且全员同步；PL 不能改底图。
- [ ] 绘制可擦除、框选删除；文本框可设字号颜色。
- [ ] 地图删除仅菜单 + 选中流程。
