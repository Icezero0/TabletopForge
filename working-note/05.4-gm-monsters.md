# Step 5 Phase 4：GM 角色创建（统一添加面板）

**目标**：GM 通过统一「添加角色」面板创建 NPC/敌人角色（`kind=npc`），与 PL 共用同一角色模型；完成 Step 5 总验收与文档同步。

**说明**：原 Phase 4「怪物专轨」（`kind=monster`、hp-change 端点）**已取消**。DB 枚举/列可保留，业务代码不再使用。

**依据**：[`05-character-core.md`](./05-character-core.md)、`docs/01` §4.2.5。

**前置**：[05.3-binding-infopanel.md](./05.3-binding-infopanel.md)（绑定、InfoPanel、上场列表）。

## 统一添加角色 UX

Popover 唯一入口「+ 添加角色」→ `AddRoomCharacterDialog`：

| 身份 | 选项 |
|------|------|
| PL | 主 PC（绑定当前玩家 `player_name`）、附加角色 |
| GM | 详细角色（6 Tab，`kind=npc`）、快速角色（名称/HP/AC/描述/Token，`kind=npc`） |

创建成功后 **Popover 保持打开**，新角色即时出现在列表；完整编辑保存回房时带 `openCharacterPopover=1` 自动重开 Popover。

**全员移除**：「仅放置 Token」、AddMonsterDialog、Popover「添加怪物」。

---

## 数据与 API

- `POST /rooms/{id}/characters`：PL `pc_main | pc_additional`；GM `npc`。
- `portrait_asset_id` 无 `token_image_asset_id` 时自动回填 token 图（详细创建头像 → Token 预览/上场）。
- **GM 添加角色（owner 为 GM）**：PL/OB 经 presenter 仅见 `damage_taken`；GM 见全量 HP/AC。
- GM `PATCH state` 降 HP 时自动累计 `damage_taken`（及 `conditions.damage_log`）。
- WS `character_state_updated`：GM-owned 角色含 `state_summary_public`（PL 用）。
