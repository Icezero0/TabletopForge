# Step 5 Phase 3：绑定、预览与 InfoPanel

**目标**：Token 与角色卡绑定；Token 上快捷显示 **HP / AC / PP**；右侧 **InfoPanel** 单槽查看完整信息；左上 **InGameCharacterList** 与「上场」流程。

**依据**：[`05-character-core.md`](./05-character-core.md)、`docs/03_frontend_design.md` §4.2.4–4.2.6、`docs/09_module_design/token_system.md` §3–5。

**前置**：[05.2-room-characters-state.md](./05.2-room-characters-state.md)（房间库 + CharacterState）。

**状态**：**已完成**（2026-06-06）。

## 数据与 API

### Token ↔ 角色绑定

- `POST/PATCH /rooms/{id}/tokens`：`linked_character_id` **必须**属于同房间 `room_characters`。
- 新建 **`POST /rooms/{id}/characters/{character_id}/spawn-token`**：
  - 从库生成 Token（默认 1 格、`width=height=grid_cell_ft`）
  - 位置：视口中心或请求体指定 x/y
  - 同一 `character_id` 可多次 spawn（多实例复用）
  - `name` 默认 character.name；可覆盖为实例名
  - Token 图：优先 `characters.token_image_asset_id`，否则首字

### 预览数据

Token 响应 embed 或 join **`state_summary`**：

```json
{ "current_hp": 12, "max_hp": 20, "ac": 16, "pp": 14 }
```

- PP 读 `characters.attributes.derived.passive_perception.value`（稳定层）。
- HP/AC 读 `character_states`。

### WebSocket

- CharacterState PATCH 后：广播 **`character_state_updated`**（payload 含 `state_summary`），前端刷新关联 Token badge 与场上列表 HP。

### 权限

- PL 移动/删除：仅绑定 **自己创建角色** 的 Token（`character.owner_id === self`）。
- GM：任意 Token（已有 `MANAGE_ANY_TOKEN`）。

## 本 Phase 做

### 后端

- Token create/update 校验 `linked_character_id` ∈ 房间库
- `spawn-token` 端点 + 权限
- Token 列表/详情返回 `state_summary`
- State 变更 WS：`character_state_updated`

### 前端

| 项 | 说明 |
|---|---|
| **Token 预览条** | `TokenItem` 底部/角标：`HP current/max · AC · PP` |
| **Token 图** | 有 `token_image` → 圆形图；无图 → 名称首字 |
| **InfoPanel** | 充实 [`InfoPanel.vue`](../frontend/src/features/table/components/InfoPanel.vue) |
| **InGameCharacterList** | 挂 `RoomPage.vue` `leftStack`；**仅场上 linked Token**（每实例一行、同角色 `#2` 编号）；HP/AC/PP、归属；点击 → InfoPanel |
| **Context Menu** | Token 菜单增加 **查看信息** |
| **底栏流程** | 「添加角色」→ **横向 Popover** 选库角色上场；末尾「+ 添加角色」→ 原创建对话框；可选 **创建后直接上场** |

#### InfoPanel 单槽

- 状态：`activeInspection: { kind: 'character', characterId, tokenId?, tokenInstanceName? } | null`
- 触发：Context Menu「查看信息」、InGameCharacterList 点击行 — **替换**旧内容（非选中自动绑定）
- 展示：稳定层分区摘要（identity/attributes/features 等）+ CharacterState 可编辑区（依 `game_role` 与 owner）
- 「完整编辑」链接 → `/characters/:id`
- PersonalMemo 不受影响

#### InGameCharacterList

- **仅场上** linked Token；同 `character_id` 多实例从第二个起显示 `#2`、`#3`…
- 列：名称、HP/AC/PP（`token.state_summary`）、归属（PC 优先 `player_name`）
- 操作：点击行 → InfoPanel（含 `tokenId` / 实例名）；**不负责上场**
- 未来可扩展先攻与回合展示
- FloatingPanel `character_list` 锚点（见 `03` §4.2.8）

#### CharacterSpawnPopover

- 底部「添加角色」按钮上方横向滚动 Popover
- 卡片：Token 图、名称、kind、玩家名、HP/AC；点击 → spawn-token（视口中心）
- 最右侧「+ 添加角色」→ `AddRoomCharacterDialog`
- PL 仅可 spawn 自己 `owner_id` 的角色；GM 全部
