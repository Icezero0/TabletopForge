## 4. Phase 5 任务范围

### 4.1 主功能：LLM 快速导入角色信息（必做）

**用户故事**：用户在界面粘贴一段自由文本（角色卡导出、论坛帖、PDF 复制等）→ 后端调用大模型提取结构化字段 → 前端以现有 6-Tab 表单预填展示 → 用户校对后写入数据库。

**技术要求**：
- 后端新增 import 端点（建议 `POST /api/v1/characters/import-preview` 或类似），**LLM 调用仅在后端**，API Key 不得暴露给前端
- 使用 **OpenAI 兼容 API**（`openai` Python SDK）
- 通过 `.env` 配置，并在 `backend/app/core/config.py` 增加 Settings 字段：
  - `LLM_API_KEY`（或 `DASHSCOPE_API_KEY`，二选一或统一别名）
  - `LLM_BASE_URL`（默认 `https://dashscope.aliyuncs.com/compatible-mode/v1`）
  - `LLM_MODEL`（默认 `qwen3.7-max` 或项目指定的 Qwen 3.x 模型名）
- 输出 JSON 须对齐 `CharacterCreate` 可接受结构：`identity` / `flavor` / `attributes` / `features` / `spells` / `equipment` / `extras`，以及 `name`、`player_name` 等顶层字段
- 参考 schema：`docs/09_module_design/character_card.md` §2；前端默认值：`frontend/src/features/character/constants.ts`
- Prompt 工程：要求模型只返回 JSON、缺失字段用空值/默认、职业名用 `DND5E_CLASSES` 枚举、数值范围合理
- 前端入口建议：
  - 角色列表页 `/characters` 增加「从文本导入」
  - 和/或 `AddRoomCharacterDialog` / `CharacterEditPage` 增加导入入口（与 `roomId` + `kind` query 兼容）
- 导入预览 → 确认 → 走现有 `createCharacter` 或 `postRoomCharacter` 流程，不另造持久化路径
- 错误处理：LLM 超时/解析失败返回可读错误；无 Key 时明确提示配置 `.env`
- **安全**：勿将 API Key 写入代码或文档；更新 `.env.development` 示例注释即可

**明确不做**（见 `05-character-core.md`）：
- 外部文件批量导入（JSON/XML 文件上传解析可留接口但不强制）
- 完整 DND5E 规则书导入、属性/技能全自动计算引擎
- 恢复 `kind=monster` 专轨

## 范围摘要（维护用）

### 主功能
添加快速导入角色信息：大文本框 → LLM 提取结构化字段 → 表单预览编辑 → 确认写入 DB。

### LLM 配置（`.env`，勿提交密钥）
```env
LLM_API_KEY=sk-xxx
LLM_BASE_URL=https://dashscope.aliyuncs.com/compatible-mode/v1
LLM_MODEL=qwen-max
```
适配 OpenAI 兼容 API；允许通过 env 切换其他模型/provider。

### 附带
1. 玩家颜色系统（房间内主色 → 绘制 / Pointer）
2. 地图资源 Popover（预览、名称、进场切换，对齐角色 Popover 体验）
3. 角色卡取消豁免/技能熟练项时自动扣除熟练加值
4. 从房间进入角色编辑详情页，返回应回到房间
